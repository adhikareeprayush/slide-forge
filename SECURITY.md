# Security Policy

## Supported versions

Security fixes are applied to the latest release on the `main` branch.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Email **adhikareeprayush@gmail.com** with:

- A description of the issue
- Steps to reproduce
- Impact assessment (if known)
- Your GitHub username (optional), if you would like credit

You should receive a response within **5 business days**. If the report is accepted, we will coordinate a fix and disclosure timeline with you.

## Scope

In scope:

- SlideForge API (`apps/api`)
- Web app (`apps/web`)
- `@slideforge/sdk`, `@slideforge/schema`, `@slideforge/ui`
- Community layout validation CI

Out of scope:

- Third-party services (NVIDIA NIM, Unsplash, Picsum)
- Misconfigured self-hosted deployments (weak secrets, open MinIO buckets)

Thank you for helping keep SlideForge safe.
